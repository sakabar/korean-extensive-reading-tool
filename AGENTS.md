# 韓国語多読ツール

## このファイルについて
- この `AGENTS.md` は日本語で記述すること。
- 実装方針、要件整理、スコープ判断、検証観点も日本語で追記・更新すること。

## 概要
- このリポジトリは、GitHub Pages で公開する韓国語多読向けの静的SPAを実装するためのものとする。
- フロントエンド構成は `React + Vite` で固定する。
- 想定ユーザーは、韓国語の課題文や読み物を貼り付けて多読する学習者とする。
- 画面に表示する文言は英語のみとする。
- 初期リリースは単一画面ツールとし、複数本文の管理機能は持たない。

## コア要件
- ユーザーは文章を1件ずつ貼り付けて利用する。
- 本文貼り付け後、少なくとも以下を即時表示すること。
  - 未知語数
    - 最初はゼロ、クリックで増えていく
  - 総語数
    - スペース区切りで算出
  - 総文字数
    - スペースは除いた文字数
  - 本文全体の未知語割合
- 読了後に少なくとも以下を表示すること。
  - 本文全体の characters per minute
  - 本文全体の words per minute
- 本文は空白単位ではなく、語彙トークン単位で扱うこと。
- ユーザーは本文中の語をワンクリックで未知語としてマークでき、再クリックで解除できること。
- 助詞、語尾、その他の機能語は未知語マーク対象から除外し、未知語割合の計算対象からも除外すること。
- 未知語としてマークした語だけを抽出して確認できるビューを用意すること。
- 未知語一覧は初期実装では同一語を集約して表示することを標準とし、出現回数は補助情報として扱える設計が望ましい。
  - 動詞・形容詞など活用語は、可能な範囲で辞書形を主表示にすること。
  - `Copy to Clipboard` ボタンを持ち、ユニーク化済み未知語の辞書形を1語1行、末尾改行付きでコピーできること。
  - コピー順は `Grouped unique words` の表示順と同一であること。
- 同一画面上にタイマー機能を持たせ、開始・停止・リセットができること。
- 本文、未知語マーク、進捗位置、タイマー関連の利用状態はブラウザストレージに保存し、再読込後も復元できること。

## 技術方針
- 実装は `React + Vite` による静的SPAで行うこと。
- 配備先は GitHub Pages とし、ビルド後の静的ファイルのみで動作する構成にすること。
- サーバー依存、アカウント機能、クラウド同期、外部API利用は前提にしないこと。
- サーバー側のフォールバック設定を必要とするルーティング構成は避けること。
- 韓国語の語彙分割・品詞判定はブラウザ内で完結させること。
- 形態素解析ライブラリの具体的なパッケージ名はこのファイルでは固定しないが、実装では以下を満たすものを選定すること。
  - ブラウザ上で動作する
  - 韓国語テキストを語彙トークン単位で扱える
  - 助詞・語尾などの除外対象を区別できる
- 少なくとも以下の状態を保持できる設計にすること。
  - 生本文
  - 解析済みトークン列
  - トークン種別（内容語 / 除外対象）
  - 未知語マーク状態
  - 最後にクリックしたトークン位置
  - タイマー状態
- 未知語割合の分母には、未知語マーク対象外のトークンを含めないこと。

## UI / 振る舞いルール
- 主な利用フローは以下とすること。
  - 本文を貼り付ける
  - ボタンを押してタイマーを開始する
  - 未知語をクリックしてマークしながら読み進める
  - ボタンを押してタイマーを停止する
  - 集計値を確認する
  - 未知語一覧を確認する
- 未知語マーク可能な語は、見た目でクリック可能だと分かるUIにすること。
- 本文表示は多読の読書感を優先し、通常時は自然な本文として見せること。
  - 単語境界を常時強調するのではなく、hover / focus / マーク時にクリック対象範囲が分かるUIを優先すること。
- ユーザーが多読時に見る本文は、紙で文章を読む時よりもヒントとなる情報量を増やさないこと。
- 助詞・語尾などの除外対象は、本文上では通常文として自然に表示し、内部的に未知語マーク対象外として扱うこと。
- 読書中は本文とタイマーを中心にし、統計確認はタイマー停止後に行う構成を優先すること。
- デスクトップとモバイルの両方で最低限支障なく操作できること。
- 辞書検索や詳細ノート機能ではなく、多読支援の軽快さを優先すること。
- 主要な画面導線は、本文貼り付け、タイマー開始、本文読解、タイマー停止、結果確認の順にたどりやすい配置にすること。
- `Single text only` のようなスコープ説明を常設UIとして置かないこと。

## v1 でスコープ外とするもの
- 複数本文の管理
- ユーザーアカウント
- クラウド保存や同期
- 外部辞書連携
- 共同編集や共有機能

## 検証観点
- 本文貼り付け時に文字数と語数が正しく更新されること。
- 助詞・語尾などの除外対象が未知語数および未知語割合に含まれないこと。
- 有効な語をクリックすると未知語マークが切り替わり、集計値が更新されること。
- 未知語としてマークした語のみを抽出表示できること。
- Unknown List の `Copy to Clipboard` で、表示順どおりの辞書形一覧が末尾改行付きでコピーできること。
- ブラウザ再読込後も、直前の本文・未知語マーク・進捗位置などが復元されること。
- タイマーの開始・停止・リセットが正しく動作すること。
- 本文更新時と Reset ボタン押下時で、タイマー初期化結果が一致すること。
- タイマー停止後に全文の CPM / WPM が確定表示されること。
- GitHub Pages のリポジトリ配下サブパスで配信しても、本番ビルドのアセット解決が壊れないこと。

## 実装方針メモ
- v1 は `React + Vite + TypeScript` の単一画面SPAとして構築する。
- GitHub Pages はリポジトリ配下サブパス配信を前提にし、`Vite` の `base` は `"/korean-extensive-reading-tool/"` に設定する。
- GitHub Actions は `CI` / `Versioning` / `Deploy Pages` の 3 workflow に分離する。
- `package.json` の `engines.node` と GitHub Actions の実行 Node は 24 系に統一する。
- `CI` は `push` / `pull_request` 時に `npm test` と `npm run build` を自動実行する。
- `Versioning` は `develop` ブランチで `workflow_dispatch` から `major` / `minor` / `patch` を選んで版数更新し、`main` に入った版数をもとに tag と GitHub Release を確定する。
- `Deploy Pages` は `main` への push ごとに GitHub Actions 公式の Pages deploy を使って `dist/` を公開する。
- `Deploy Pages` の初回実行前に、GitHub の `Settings > Pages` で `Build and deployment > Source` を `GitHub Actions` に設定して Pages site を有効化しておくこと。
- favicon は `bunbougu_marker.png` を元画像として `public/favicon.ico` と `public/favicon-32x32.png` を配置し、`index.html` から相対参照で読み込む。
- favicon 用画像は縦横比を維持したまま透明余白付きの正方形に収め、小サイズでもペン先と本体の判別が残るようにする。
- 韓国語の語彙トークン化と品詞判定は `oktjs` を利用してブラウザ内で完結させる。
  - `Open Korean Text` 系の解析結果を使い、少なくとも `Josa / Eomi / PreEomi / Conjunction / Modifier / VerbPrefix / Suffix / Space / Punctuation / Others / KoreanParticle` などは除外対象として扱う。
  - `oktjs` は初期 bundle へ直載せせず、本文解析時に lazy load して専用 chunk に分離する。
- 状態管理は単一のアプリ状態で扱い、少なくとも以下を `localStorage` に保存する。
  - 生本文
  - 解析済みトークン列
  - 未知語マーク状態
  - 最後にクリックしたトークンID
  - タイマー状態
- 本文編集時は再解析を優先し、未知語マークと進捗位置はリセットする。
- 本文編集時のタイマーリセットと Reset ボタン押下時のタイマーリセットは、同一ロジックを共有すること。
- 未知語一覧は `normalizedSurface` 単位で集約し、出現回数を補助表示する。
- 未知語一覧の主表示は、活用語では辞書形を優先する。
- タイマーは `baseElapsedMs / isRunning / lastStartedAt` で復元し、再読込時に実経過時間を再計算する。
- 全文の CPM / WPM は、タイマー停止後の確定値として表示する。

## 要件整理メモ
- 総語数は本文の空白区切りで計算する。
- 本文表示と未知語マーク操作は空白区切りではなく形態素トークン単位で行う。
- 本文表示は自然文ベースとし、クリック判定だけを語彙トークン単位で保持する。
- 内部的な品詞やクリック可否を持っていても、通常表示では学習者に不要な区別を見せない。
- 未知語割合の分母には `isMarkable` な内容語トークンだけを含め、助詞・語尾・記号は含めない。
- 表示文言は英語のみとし、日本語は設計文書と実装メモに限定する。

## スコープ判断メモ
- v1 では本文の複数管理、辞書連携、アカウント、共有機能は入れない。
- 解析器は差し替え可能な薄い関数層の内側に閉じ込めるが、実装上は `oktjs` を標準採用とする。
- 本文変更後に旧マーク状態をトークン対応付けで引き継ぐ機能は v1 に入れない。

## 実施済み検証メモ
- `npm test` で単体テストとUIテストを通し、未知語集計、進捗計算、永続化、タイマー復元を確認した。
- `npm run build` で本番ビルドを通し、GitHub Pages 向け `base` 設定込みで静的出力できることを確認した。
- `npm run build` で `oktjs` が初期 entry chunk から分離され、chunk size warning が出ないことを確認対象とする。
- GitHub Actions の 3 workflow 構成追加後に、ローカルでも `npm test` と `npm run build` が継続して通ることを確認する。
- Node 24 への統一後に、`package.json` の `engines.node` と 3 workflow の `node-version` がすべて 24 系で一致していることを確認する。
- `Versioning` は `develop` で manual dispatch し、`main` 反映後に同一 version の tag / Release が二重作成されないことを確認対象とする。
- `Deploy Pages` は GitHub Pages の公開元を GitHub Actions に設定した上で、`main` push で `dist/` が公開されることを確認対象とする。
- `Deploy Pages` 実行時に `actions/configure-pages` が `Get Pages site failed` または `Not Found` で失敗した場合は、repository の Pages が未有効化か `Source` 未設定を疑うことを確認メモとして残す。
- favicon の生成後に `npm run build` を通し、`dist/` に `favicon.ico` と PNG が出力され、`dist/index.html` から相対参照で解決されることを確認した。
